## 1. 问题框定

**一句话重述**: 用现成的现代前端 UI 模板替换 Markwritter 的前端基础架构，获得更完善的布局系统、主题系统、Auth 页面和 Error 页面，同时保持全部业务功能不变。

**角色**: 开发者
**触发条件**: 前端框架需更换，要求现代、非过时、现成可用的 UI Web 框架
**预期结果**: 基于 shadcn-dashboard-landing-template 的 Markwritter Web 应用，保持全部 7 个业务页面功能正常

**已确认的决策**:
- ✅ 保持 FastAPI 后端不变
- ✅ 保持 Vite SPA 架构（不引入 SSR）
- ✅ 选择方案 A: shadcn-dashboard-landing-template 的 Vite 版本

**非目标**:
- 不改变后端 (FastAPI) 架构
- 不引入 SSR/SSG
- 不迁移到 Next.js（无 SSR/SEO 需求）
- 不新增功能

## 3. 模板选型分析

### 3.1 候选模板对比

| 模板 | Stars | Chat 支持 | Vite 版 | 技术栈匹配度 | 选择 |
|---|---|---|---|---|---|
| **shadcn-dashboard-landing-template** | 689 | ✅ 内置 Chat demo | ✅ 有 | ⭐⭐⭐⭐⭐ | **✅ 选定** |
| next-shadcn-admin-dashboard | 2,155 | ❌ Coming Soon | ❌ 仅 Next.js | ⭐⭐⭐ | 排除 |
| horizon-ui/shadcn-nextjs-boilerplate | 632 | ✅ ChatGPT UI | ❌ 仅 Next.js | ⭐⭐ | 排除 |
| shadboard | 647 | ✅ Chat App | ❌ 仅 Next.js | ⭐⭐⭐ | 排除 |
| vercel/chatbot | 20,210 | ✅ 生产级 | ❌ 仅 Next.js | ⭐⭐ | 排除 |

### 3.2 选定理由

**shadcn-dashboard-landing-template Vite 版** 的优势:
1. **唯一提供 Vite SPA 版本的主流模板** — 无需改变构建和部署流程
2. **技术栈完全匹配** — React 19, Vite, Tailwind v4, Zustand, shadcn/ui, Radix UI
3. **内置 Chat demo** — 可作为 Chat 页面的 UI 起点
4. **30+ 页面** — 包含 Auth (登录/注册/忘记密码)、Error (404/401/403/500)、Settings、Pricing 等
5. **实时主题定制器** — tweakcn 集成，支持 7 种预设主题
6. **MIT 许可** — 可商用
7. **2026-04-30 仍在活跃维护**
